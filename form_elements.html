- - -

<button ... disabled="disabled">

Usage Guidelines

    Blue buttons are for primary actions. There should be only one primary action on each page.

    Primary buttons should always be on the right side of secondary buttons.

    Disabled/Inactive buttons should have a cursor: not-allowed styling.

    Button links should always be blue.

    Accordion headers should be an h3 tag. Any headers inside the accordion need an h4.

---------
Typeahead
<ul-typeahead ul-label="Local Source"
              ng-model="ctrl.local.selected"
              tph-search="ctrl.search($search)"
              tph-label="it.FamilyName + ', ' + it.GivenName" />

Usage Guidelines

    Drop downs are difficult for users when the options exceed 15 or more. Ensure they are able to search, categorize options or opt for typeahead functionality to help users select the best option quickly.

    Multi-select and typeahead elements need placeholders to help users.

    All drop downs, multi-select and typeahead elements need a label.

Checkboxes

example

Sample Code

<div class="btn-group" data-toggle="buttons">
 <input type="checkbox" autocomplete="off"> Checkbox 1
 <input type="checkbox" autocomplete="off"> Checkbox 2
 <input type="checkbox" autocomplete="off"> Checkbox 3
</div>

Usage Guidelines

    Checkboxes are used when there are lists of options and the user may select any number of choices, including zero, one, or several.

    A stand-alone checkbox is used for a single option that the user can turn on or off.

Radio Buttons

example

Sample Code

<div class="radio">
 <label><input type="radio" name="optradio">Option 1</label>
</div>
<div class="radio">
 <label><input type="radio" name="optradio">Option 2</label>
</div>
------
Disabled
<div class="radio disabled">
 <label><input type="radio" name="optradio" disabled>Option 3</label>
</div>

Usage Guidelines

    Radio buttons are used when there is a list with only one mutually exclusive choice in the UI.

    Do not allow the user to select multiple options in a radio button list.

    Radio button toggles are a great alternative to small drop down lists in mobile-view.

    Radio buttons should come with a default choice.

Footer Action Bar

example

Sample Code

<footer class="footer fullscreen-footer noprint">
 <div class="container">
  <div class="row">
   <div class="col-md-12 col-sm-12">
    <div class="btn-bar-left pull-left">
     <button type="button" class="btn btn-default">Secondary</button>
    </div>

    <div class="btn-bar-right pull-right">
     <button type="button" class="btn btn-link">Cancel</button>
     <button type="button" class="btn btn-primary">Save</button>
    </div>
   </div>
  </div>
 </div>
</footer>

Usage Guidelines

    Action footer component is used on form CRUD screens throughout PureOHS.

    Each footer action bar should have one primary action button, a secondary action button/link in the right-hand corner. Any additionaly secondary actions should be located in the left-hand corner.

    Icon buttons should be used sparingly in the footer action bar.

Sample Code

<div class="alert alert-success" role="alert"><i class="fa fa-check" aria-hidden="true"></i>Success! Your information has been saved!</div>

<div class="alert alert-warning" role="alert"><i class="fa fa-warning" aria-hidden="true"></i> Careful! You might be making a mistake...</div>

<div class="alert alert-danger" role="alert"><i class="fa fa-exclamation" aria-hidden="true"></i> Uh oh! You might want to check this out...</div>

Usage Guidelines

    Alert messages should be clearly visible, explicit, understandable, polite and constructive in how to handle the problem.
